联通邮政商盟触点系统
==========

接口设计文档（修改版 V2.0）
----------------

**Base URL**: `/api/v1`  
**认证方式**: JWT Bearer Token（Header: `Authorization: Bearer <token>`）  
**统一响应格式**:
    {
      "code": 200,
      "msg": "success",
      "data": { ... }
    }

**分页响应格式**:
    {
      "total": 100,
      "pageNo": 1,
      "pageSize": 20,
      "list": [ ... ]
    }

一、认证模块 `/auth`
--------------

### 1.1 用户登录

    POST /auth/login
    Content-Type: application/json
    
    请求体:
    {
      "username": "13900001111",
      "password": "abc123456"
    }
    
    响应体:
    {
      "code": 200,
      "msg": "success",
      "data": {
        "token": "eyJhbGciOiJIUzI1NiIsInR...",
        "userId": 1,
        "username": "admin001",
        "realName": "张三",
        "phone": "13900001234",
        "roles": ["ROLE_CITY"],                  // 【修复1】改为数组，支持多角色
        "dataScopeType": "CITY",
        "scopeCityId": 2,
        "scopeOutletId": null,
        "lastLoginAt": "2026-06-20 14:30:00"
      }
    }
    
    错误案例:
    {
      "code": 401,
      "msg": "用户名或密码错误"
    }
    
    {
      "code": 401,
      "msg": "账号已被禁用，请联系管理员"
    }

### 1.2 用户登出

    POST /auth/logout
    Authorization: Bearer <token>
    
    响应体:
    {
      "code": 200,
      "msg": "登出成功"
    }

### 1.3 修改密码

    PUT /auth/password
    Authorization: Bearer <token>
    Content-Type: application/json
    
    请求体:
    {
      "oldPassword": "abc123456",
      "newPassword": "def789012",
      "confirmPassword": "def789012"
    }
    
    响应体:
    {
      "code": 200,
      "msg": "密码修改成功，请重新登录"
    }
    
    错误案例:
    {
      "code": 400,
      "msg": "旧密码不正确"
    }
    
    {
      "code": 400,
      "msg": "新密码不符合安全规范（至少8位，包含大小写字母、数字、特殊符号）"
    }

### 1.4 获取当前用户信息

    GET /auth/me
    Authorization: Bearer <token>
    
    响应体:
    {
      "code": 200,
      "msg": "success",
      "data": {
        "userId": 1,
        "username": "admin001",
        "realName": "张三",
        "phone": "13900001234",
        "roles": ["ROLE_CITY"],
        "dataScopeType": "CITY",
        "scopeCityId": 2,
        "menu": [ ... ]  // 根据角色返回动态菜单
      }
    }

### 1.5 重置密码（管理员操作）【新增，支持问题9修复】

    POST /auth/reset-password/{userId}
    Authorization: Bearer <token>
    Content-Type: application/json
    
    说明: 仅PROVINCE角色可操作，用于管理员为发展人重置初始密码
    
    请求体:
    {
      "resetType": "AUTO"  // AUTO=自动生成 | MANUAL=手动指定
    }
    
    响应体（AUTO方式）:
    {
      "code": 200,
      "msg": "密码已重置并发送短信",
      "data": {
        "userId": 123,
        "username": "13900001111",
        "tempPassword": "aB@12345XyZ9",  // 仅在此次响应显示一次
        "smsStatus": "sent",
        "smsPhone": "13900001111",
        "expiryTime": "2026-06-21 14:30:00"  // 临时密码24小时内有效
      }
    }
    
    响应体（MANUAL方式）:
    {
      "code": 200,
      "msg": "密码已设置",
      "data": {
        "userId": 123,
        "username": "13900001111",
        "passwordHash": "***"  // 不返回密码本身
      }
    }
    
    错误案例:
    {
      "code": 403,
      "msg": "仅超级管理员可重置他人密码"
    }
    
    {
      "code": 400,
      "msg": "该用户不存在或已禁用"
    }
    
    {
      "code": 500,
      "msg": "短信发送失败，请稍后重试"
    }

### 4.4 审核申请（多级）【修复8：支持多级审核】

    PUT /developer-applies/{applyId}/audit
    Authorization: Bearer <token>
    Content-Type: application/json
    
    权限:
    - OUTLET: 仅能审核本网点申请（OUTLET级别）
    - CITY: 仅能审核本市申请（CITY级别）
    - PROVINCE: 仅能审核省级申请（PROVINCE级别）
    
    请求体:
    {
      "auditLevel": "CITY",  // 当前操作人审核的级别
      "status": "APPROVED",  // APPROVED | REJECTED
      "auditRemark": "符合条件，待省级最终审核"
    }
    
    说明:
    1. 网点管理员审核通过 → 状态变为 OUTLET_APPROVED，自动转推到地市
    2. 地市管理员审核通过 → 状态变为 CITY_APPROVED，自动转推到省级
    3. 省级审核通过 → 状态变为 PROVINCE_APPROVED
       后端事务执行:
       ① 生成随机密码（12位，大小写+数字+特殊符号）
       ② INSERT INTO sys_user { username=phone, password_hash=BCrypt(密码), 
                                  role=ROLE_DEVELOPER, data_scope_type=OUTLET, 
                                  scope_outlet_id=outlet_id }
       ③ INSERT INTO biz_developer { user_id=新账号, apply_id=本申请, ... }
       ④ UPDATE biz_developer_apply SET user_id=新账号, status=PROVINCE_APPROVED
       ⑤ 通过短信网关发送初始密码给申请人
    4. 任何级别拒绝 → 状态变为 REJECTED，流程终止
    
    响应体:
    {
      "code": 200,
      "msg": "审核操作成功",
      "data": {
        "applyId": 1001,
        "status": "CITY_APPROVED",
        "nextAuditor": {
          "level": "PROVINCE",
          "role": "ROLE_PROVINCE",
          "needsReview": true
        },
        "tempPassword": null,  // 仅省级通过时才返回（不安全，直接发短信）
        "smsStatus": "sent"
      }
    }
    
    错误案例（拒绝）:
    {
      "code": 200,
      "msg": "审核操作成功（拒绝）",
      "data": {
        "applyId": 1001,
        "status": "REJECTED",
        "rejectReason": "不符合条件"
      }
    }
    
    {
      "code": 403,
      "msg": "您没有权限审核该申请"
    }
    
    {
      "code": 400,
      "msg": "申请已处理，请勿重复操作"
    }
    
    {
      "code": 400,
      "msg": "申请地市与网点地市不匹配，审核已拒绝"
    }

----------------------------

### 5.1 提交发展记录（意向单）

    POST /orders
    Authorization: Bearer <token>
    Content-Type: application/json
    
    权限: OUTLET | DEVELOPER
    
    请求体:
    {
      "businessType": "MOBILE_CARD",  // MOBILE_CARD | BROADBAND | OTHER
      "developSource": "SITE_USER",   // SITE_USER | OUTLET_SELF
      "customerName": "陈XX",
      "customerPhone": "15900001111",
      "customerIdCard": "360102199506062222",
      "customerAddress": "南昌市青云谱区XX路",
      "remark": "客户有宽带需求"
    }
    
    说明:
    - 当前用户的 outlet_id 和 developer_id 从scope自动获取
    - 发展人提交: develop_source 自动设为 SITE_USER
    - 网点管理员提交: 可选择 SITE_USER 或 OUTLET_SELF
    
    响应体:
    {
      "code": 200,
      "msg": "发展记录提交成功",
      "data": {
        "orderId": 5001,
        "orderNo": "BIZ20260620000001",
        "customerPhone": "15900001111",
        "leadStatus": "PENDING",
        "isDuplicate": false,  // 【修复7】去重提示
        "duplicateOrderId": null,
        "createdAt": "2026-06-20 14:30:00"
      }
    }
    
    错误案例:

    POST /commission-rules
    Authorization: Bearer <token>
    Content-Type: application/json

    {

      "platformRatio": 0.4000,
      "effectiveDate": "2024-01-01",
      "expiryDate": null,
      "remark": "移网卡引流全归网点"
    }
    
    校验:
    - outletRatio + developerRatio + platformRatio = 1.0000（精确到4位小数）
    - effectiveDate <= 当前日期（新规则立即生效）或 > 当前日期（定时生效）
    - 不允许重复规则（business_type + develop_source + commission_phase + effective_date 唯一）
    
    响应体:
    {
      "code": 200,
      "msg": "规则创建成功",
      "data": {
        "id": 1,
        "ruleName": "移网业务·发展人发展·引流佣金-2024",
        "createdAt": "2026-06-20 14:30:00"
      }
    }
    
    错误案例:
    {
      "code": 400,
      "msg": "比例之和必须等于1.0000，当前和为0.9999"
    }
    
    {
      "code": 400,
      "msg": "该规则已存在（移网 + 发展人发展 + 引流 + 2024-01-01）"

### 12.1 多级审核流程（State Machine）

    // 审核状态机示例
    public enum DeveloperApplyStatus {
        PENDING("待审核"),
        OUTLET_APPROVED("网点通过"),
        CITY_APPROVED("市级通过"),
        PROVINCE_APPROVED("省级通过"),
        REJECTED("已拒绝");
    
        // 允许的状态转移
        public static boolean isValidTransition(
            DeveloperApplyStatus from, 
            DeveloperApplyStatus to) {
    
            if (from == PENDING) {
                return to == OUTLET_APPROVED || to == REJECTED;
            } else if (from == OUTLET_APPROVED) {
                return to == CITY_APPROVED || to == REJECTED;
            } else if (from == CITY_APPROVED) {
                return to == PROVINCE_APPROVED || to == REJECTED;
            }
            return false;
        }
    }

### 12.3 随机密码生成

    public String generateRandomPassword() {
        String uppercase = "ABCDEFGHIJKLMNOPQRSTUVWXYZ";
        String lowercase = "abcdefghijklmnopqrstuvwxyz";
        String digits = "0123456789";
        String special = "!@#$%^&*";
        String all = uppercase + lowercase + digits + special;
    
        Random random = new Random();
        StringBuilder password = new StringBuilder();
    
        // 确保至少包含各类字符
        password.append(uppercase.charAt(random.nextInt(uppercase.length())));
        password.append(lowercase.charAt(random.nextInt(lowercase.length())));
        password.append(digits.charAt(random.nextInt(digits.length())));
        password.append(special.charAt(random.nextInt(special.length())));
    
        // 剩余8位从全集中随机选择
        for (int i = 0; i < 8; i++) {
            password.append(all.charAt(random.nextInt(all.length())));
        }
    
        // 随机打乱顺序
        return password.toString().split("")[0];
    }

十二、业务审核日志 `/audit-logs`【新增，V2.1】
---------------------------------

### 12.1 审核日志说明

系统使用 `biz_audit_log` 表记录所有多级审核操作，包括发展人申请审核和业务订单审核。

**审核日志表结构**:

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| target_type | VARCHAR(30) | DEVELOPER_APPLY / ORDER |
| target_id | BIGINT | 申请ID或订单ID |
| audit_level | VARCHAR(20) | OUTLET / CITY / PROVINCE |
| audit_phase | VARCHAR(10) | LEAD / FORMAL（仅订单使用） |
| auditor_id | BIGINT | 审核人ID |
| auditor_name | VARCHAR(50) | 审核人姓名 |
| audit_status | VARCHAR(20) | APPROVED / REJECTED |
| audit_remark | VARCHAR(500) | 审核备注 |
| audit_time | DATETIME | 审核时间 |
| created_at | DATETIME | 记录创建时间 |

**说明**:
- 每次审核操作（通过或拒绝）都会自动记录一条审核日志
- 发展人申请列表和详情接口的 `auditLog` / `auditHistory` 字段从该表实时查询
- 订单列表和详情接口的 `leadAuditLog` / `formalAuditLog` 字段从该表实时查询
- 审核日志不可修改或删除，保证审计追溯完整性
